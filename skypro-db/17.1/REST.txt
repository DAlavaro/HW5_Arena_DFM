RESR
(Representational State Transfer)

Парадигма, архитектурный стиль.
Передача сомоописываемого состояния

- Один из способов организации API
- Основан на HTTP
- Рекомендации, а не спецификация

Сервис, который реализует REST


В чем стиль?
 - Клиент серверное взаимодействие
 - Единообразие интерфейсов
 - Без состояния
 - Кешируемость ( Приложение должно имееть возможность реализации КЭША)
 - Слоеная система


Вспоминаем методы и коды ответов HTTP

POST    PUT         OPTIONS
GET     PATCH       TRACE
        DELETE      HEAD

РЕСУРС или эндпоинт
строка определяющая URI

Правила REST

Правило 1 Множественное число
не правильно        /user/user/1
правильно           /users/users/1

Правило 2 Никаких глаголов(get, set, add)

Правило 3 Только строчные буквы (lowercase)

Правило 4 Короткие URL
